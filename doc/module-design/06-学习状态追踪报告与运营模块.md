# 学习状态追踪报告与运营模块详细设计

## 1. 模块目标

本模块负责将评估、训练、AI 洞察等事件沉淀为稳定的学习状态，并输出学生成长视图、家长报告和运营看板。

核心目标：

1. 构建学生掌握度状态
2. 识别风险信号
3. 生成日报、周报和热力图
4. 为运营后台提供可查询指标

---

## 2. 逻辑边界

### 2.1 本模块负责

1. 事件采集
2. 学习状态投影
3. 掌握度快照
4. 风险检测
5. 报告生成
6. 运营分析查询

### 2.2 本模块不负责

1. 评估会话本身
2. 训练流程本身
3. AI 模型调用
4. 教材和题库维护

```ts
class StudentMasterySnapshot {
  id: string;
  studentId: string;
  subject: 'chinese' | 'math' | 'english';
  knowledgePointId: string;
  masteryScore: number;
  confidenceScore: number;

  type: 'streak_break' | 'retry_failure' | 'mastery_drop' | 'high_hint_dependency';
  level: 'low' | 'medium' | 'high';
  summary: string;
}
```

## 5. 核心流程

## 5.1 学习状态更新流程

```mermaid
flowchart TD
    A[接收 assessment.completed 或 mission.completed] --> B[解析知识点结果]
    B --> C[更新掌握度快照]
    C --> D[更新错误类型统计]
    D --> E[运行风险检测]
    E --> F[写入学习状态投影]
```

## 5.2 周报生成流程

```mermaid
flowchart TD
    A[到达周报生成时点] --> B[读取一周学习事件]
    B --> C[汇总掌握度变化]
    C --> D[汇总任务完成与中断情况]
    D --> E[汇总风险与恢复事件]
    E --> F[生成周报结构]
    F --> G[发送给 AI 解释模块]
    G --> H[保存周报]
```

## 8. 状态计算规则

### 8.1 掌握度建议

掌握度更新参考输入：

1. 最近评估分数
2. 最近训练正确率
3. 错题回测结果
4. 提示依赖程度

### 8.2 风险信号建议

以下情况触发风险检测：

1. 连续 3 天未完成任务
2. 同一知识点连续回测失败
3. 提示依赖率持续过高
4. 掌握度明显回落

1. 学习状态可随评估和训练事件更新
2. 掌握度热力图可查询
3. 日报和周报可生成
4. 家长风险提醒可读取
5. 运营后台能看到核心指标
