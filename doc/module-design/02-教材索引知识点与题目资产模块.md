# 教材索引知识点与题目资产模块详细设计

## 1. 模块目标

本模块负责建立系统的内容底座，确保所有评估、训练、AI 分析都基于统一教材索引和知识点资产运行。

核心目标：

1. 导入语文、数学、英语教材索引
2. 维护教材章节树
3. 维护知识点树和前置关系
4. 维护题目资产与知识点映射

---

## 2. 逻辑边界

### 2.1 本模块负责

1. 教材目录扫描
2. 教材文件元数据入库
3. 单元、课时结构维护
4. 知识点树维护
5. 题目录入、审核和发布
6. 知识点与题目映射

### 2.2 本模块不负责

1. 个性化评估结果
2. 训练任务会话
3. 学生掌握度计算
4. AI 助教对话

## 5. 核心流程

## 5.1 教材导入流程

```mermaid
flowchart TD
    A[扫描教材目录] --> B[识别文件元数据]
    B --> C[创建教材册次记录]
    C --> D[抽取目录页]
    D --> E[生成章节草稿]
    E --> F[运营人工校对]
    F --> G[发布教材结构]
```

## 5.2 知识点挂载流程

```mermaid
flowchart TD
    A[创建知识点] --> B[设置父级知识点]
    B --> C[设置前置依赖]
    C --> D[挂载到教材课时]
    D --> E[绑定题目]
    E --> F[发布]
```

## 9. 逻辑规则

1. 题目发布前必须至少绑定 1 个知识点
2. 课时发布前必须归属单元
3. 同一知识点可绑定多题，题型需覆盖基础训练和评估场景
4. 题目难度分级统一使用 1 到 5
5. 任何教材结构自动抽取结果在发布前都必须人工确认

## 11. 测试要点

1. 同一教材文件重复导入必须幂等
2. 教材树查询必须按单元和课时稳定排序
3. 未发布题目不可被评估和训练模块消费
4. 知识点依赖不能形成环
5. 删除知识点前必须校验是否仍有发布题目绑定

---

## 12. 模块完成定义

满足以下条件视为模块完成：

1. 三科教材册次可导入
2. 教材树可查询
3. 知识点树可维护
4. 题库可维护并发布
5. 评估和训练模块可通过稳定接口获取内容资产
