# 评估引擎模块详细设计

## 1. 模块目标

本模块负责将教材、知识点与题库资产编排成正式评估流程，并输出可用于后续学习计划和训练编排的结构化结论。

核心目标：

1. 支持入门诊断、单元评估、阶段评估、微评估和错题回测。
2. 支持多题型统一答题与判定。
3. 支持客观题规则判定与主观题、过程题 AI 分析。
4. 生成知识点层级的评估结果。
5. 发布评估完成事件供后续模块消费。

---

## 2. 逻辑边界

### 2.1 本模块负责

1. 评估定义和组卷。
2. 评估会话创建。
3. 答题记录。
4. 判题与结论汇总。
5. 评估结果页数据。

### 2.2 本模块不负责

1. 教材和知识点主数据维护。
2. 富媒体题目编辑和导入。
3. 今日任务生成。
4. 长周期掌握度投影。
5. 家长周报。

## 5. 核心流程

## 5.1 发起评估流程

```mermaid
flowchart TD
    A[接收评估请求] --> B[加载学生学科配置]
    B --> C[加载评估定义]
    C --> D[从题库选择已发布题目]
    D --> E[加载 QuestionRenderPayload 与 AnswerSchema]
    E --> F[创建评估会话]
    F --> G[返回评估纸]
```

## 5.2 答题与判定流程

```mermaid
flowchart TD
    A[提交答案] --> B[校验答案 schema]
    B --> C{是否客观题}
    C -->|是| D[规则判题]
    C -->|否| E[调用 AI 分析]
    D --> F[保存题目结果]
    E --> F
    F --> G[更新会话进度]
    G --> H{是否全部完成}
    H -->|否| I[返回下一题]
    H -->|是| J[汇总评估结果]
    J --> K[发布 assessment.completed]
```

## 7. 内部接口与依赖

本模块依赖：

1. `QuestionRuntimePort`
2. `AIAnalysisPort`
3. `EventBus`

接口建议：

```ts
interface QuestionRuntimePort {
  pickAssessmentQuestions(query: AssessmentQuestionQuery): Promise<QuestionRuntimeAsset[]>;
  getQuestionRuntime(questionId: string): Promise<QuestionRuntimeAsset>;
}

interface AIAnalysisPort {
  analyzeAssessmentAnswer(command: AnalyzeSubjectiveAnswerCommand): Promise<AIGradeResult>;
}
```

说明：

1. `QuestionRuntimeAsset` 由 `08` 模块提供，内含题目渲染数据、作答模式和判题配置。
2. 评估引擎不直接依赖题目编辑表结构。

## 9. 逻辑规则

1. 评估会话完成后不可再次提交答案。
2. 同一题目只能保留一份最终判定结果。
3. 评估只允许使用已发布、许可合法的题目。
4. AI 判定结果必须保存原始摘要和结构化结论。
5. 微评估题量必须受时长限制。
6. 错题回测优先使用原题或同知识点同难度变体。
7. AI 分析失败时必须返回明确错误并进入异常处理，不输出假结果。

## 11. 测试要点

1. 评估组卷必须只使用已发布题目。
2. 客观题结果必须可重复计算。
3. 公式题、图文题和子题组题都能正常提交答案。
4. 主观题 AI 分析失败时必须返回可识别错误。
5. 中途退出评估会话必须有明确状态。
6. 结果页必须能定位到知识点层级。

---

## 12. 模块完成定义

满足以下条件视为模块完成：

1. 五类评估会话均可创建。
2. 多题型答题和判定主链路可跑通。
3. 客观题与主观题都可得出结果或明确错误。
4. 评估完成事件可被其他模块消费。
5. 结果页数据结构稳定且可联调。
